# 📑 Tab Board

实时 Chrome 标签页看板 —— 把挤成一行、看不清的几十个 tab，自动按语义归类成一屏可读的卡片面板。在一个页面里完成阅读、跳转、关闭、收藏、批量整理，并自带永久收藏夹、暂存区、定期清理提醒和自动备份。

> **仅支持 macOS + Google Chrome**（通过 AppleScript 读取并控制本机 Chrome）。纯 Python 标准库，无第三方依赖。

## 快速开始

**直接运行（开发 / 日常用）：**
```bash
python3 ~/Desktop/tabboard/tab_board.py
```
启动后自动打开浏览器 `http://localhost:8765`。`Ctrl+C` 停止。

**安装成全局命令（可选，一次性）：**
```bash
pipx install ~/Desktop/tabboard          # 或 pipx install git+<你的GitHub仓库>
tabboard                                  # 之后任何目录敲这一个词即可
```
> 用 `pipx install --editable ~/Desktop/tabboard` 安装，则改源码后命令即时生效，无需重装。

**端口被占用时**：再次运行会检测到“已在运行”，直接打开浏览器而不报错。

**演示 / 试用（不读真实 Chrome）：**
```bash
python3 ~/Desktop/tabboard/tab_board.py --demo   # 或 tabboard --demo
```
用一组预置假标签页展示全部功能，**不读取真实 Chrome、不读写任何真实数据文件**。适合录屏、截图、或没有 Mac/Chrome 时快速体验。顶部会显示「🎬 DEMO」徽标。

## 核心概念：tab 的三种归宿

理解这三者的区别，是用好这个工具的关键：

| | ✕ 关闭 | 📥 暂存 | ⭐ 收藏 |
|---|---|---|---|
| **场景** | 不要了 | 马上还会用，先收起 | 长期参考，随时要找到（如周会纪要） |
| **tab 是否关闭** | 关 | 关 | **不关**（你自己决定何时关） |
| **再打开后** | —— | **消失**（回到实时 tab，任务完成） | **不消失**（永久留在收藏夹） |
| **存储** | —— | 暂存区（临时中转） | 收藏夹（永久库，可视化分类） |

```
┌─────────────────────────────────────────────────────────────┐
│ 📑 Tab Board   44 tabs · 13 类 · 📦 2 暂存    [🔍搜索] [📚收藏夹] [🧹去重] [↻] │  ← 顶栏
├─────────────────────────────────────────────────────────────┤
│ (收藏夹视图下，这里会出现周清理提醒条)                            │  ← 通知区
├──────────────────┬──────────────────┬───────────────────────┤
│ 📦 暂存区          │ 📊 Dashboards     │ 🎓 教育项目            │
│ 🤝 1:1 Meeting    │ 🧭 Onboarding     │ 📐 PRD & 治理参考      │  ← 独立列卡片
│   ⠿ ▾ 标题  ⭐📥✕  │   ...             │   ...                 │
└──────────────────┴──────────────────┴───────────────────────┘
```

- **两个视图**，顶部「📚 收藏夹」按钮切换：
  - **实时 tab 视图**（默认）：当前 Chrome 所有 tab + 暂存区
  - **收藏夹视图**：永久收藏库
- **每 2 秒自动刷新**，内容无变化时不重绘（不闪烁）；开关任何 tab 都会自动反映。
- **固定独立列布局**：折叠 / 改动某个模块只影响所在列，其它列纹丝不动。

## 功能详解

### 1. 实时分组看板

- 通过 AppleScript 实时读取 Chrome 所有窗口/标签（标题 + URL + 窗口 id + tab id）。
- 按关键词规则自动归到 13 个分类（见下），认不出的进「📦 其他 / 未分类」。
- 标题自动清洗（去掉 ` - Feishu Docs` 后缀、隐藏水印字符）。
- 看板自身那个 tab（localhost:8765）会被自动隐藏，不出现在列表里。
- 顶栏状态显示：`X tabs · Y 类 · 📦 Z 暂存`。

**13 个分类**：🤝 1:1 Meeting Notes、📊 Dashboards & 数据、🧭 Onboarding & 知识库、🎓 教育项目、🤖 AI 学习课程、📐 PRD & 治理产品参考、🔑 Access & Playbook、📋 Meeting Notes & OKR、🛠️ Support / 工具、🏢 Office / 生活、✉️ Personal · Email & YouTube、🎉 Personal · 生日 & Shopping、📦 其他。

### 2. 单个 tab 操作

每个 tab 条目，鼠标悬停时行尾出现操作按钮：

| 操作 | 图标 | 说明 |
|---|---|---|
| **跳转** | 点标题 | 激活并置顶 Chrome 里那个真实 tab（不新开） |
| **收藏** | ⭐ | 存入收藏夹，**不关闭**。已收藏的 ⭐ 金色常亮；再点一次=取消收藏 |
| **暂存** | 📥 | 关闭 tab 但记住它，进暂存区 |
| **关闭** | ✕ | 关闭这个真实 tab |

> ⭐ 金色常亮 = 已在收藏夹，所以一眼能看出哪些 tab 已经收藏过。

### 3. 自动归类 & 拖动改归类

- **自动**：按 `RULES`（标题 + URL 关键词）和 `HOST_RULES`（域名兜底）归类，先匹配先归。
- **手动纠正**：当某个 tab 归类不准，**把它拖到另一个分类卡片上**即可。
  - 拖动时目标分类显示**蓝色虚线框**（= 并入这个分类）。
  - 归类按 **URL** 记住，存到 `~/.tabboard_overrides.json`，**重启、明天重开同一文档都记得**。
  - 覆盖优先级高于关键词规则（你的判断 > 机器的猜测）。

### 4. 折叠 / 展开

- **点分类标题栏任意位置**（手柄 ⠿ 除外）→ 折叠 / 展开该组。
- 标题左侧小三角 ▾ 旋转为 ▸ 表示已折叠，内容平滑滑动收起（`grid-template-rows` 动画）。
- 折叠状态存浏览器 `localStorage`（key `tb_collapsed`）：**自动刷新不会弹回、刷新页面也记得**。

### 5. 拖动手柄重排分组

- 每个分类标题最左侧有 **⠿ 手柄**（鼠标变手型）。
- **抓手柄拖动整个模块** → 重排分类顺序，拖影是整个模块。
- 放下时显示**一根发光的插入线**：悬在目标**上半部**→线在其上方（插到前面）；**下半部**→线在下方（插到后面）。
- 顺序存 `localStorage`（key `tb_order`），刷新后保持。
- 暂存区固定置顶、收藏夹视图不参与手柄重排（这两类无手柄）。

### 6. 搜索 / 去重 / 整组关闭

- **🔍 搜索框**：实时按标题过滤，空的分类自动隐藏。两个视图都生效。
- **🧹 一键去重**：关闭 URL 重复的 tab（每个 URL 保留第一个），弹窗告知关了几个。
- **关闭整组 ✕**（每个分类标题右侧）：二次确认后，关闭该分类下所有真实 tab。
- **↻ 刷新**：强制立即重新读取。

### 7. 📦 暂存区（临时收起）

“现在用不到、马上还要用、怕关了找不到”的 tab 用这个。

- 在实时 tab 上点 **📥** → tab 关闭，进入**置顶的 📦 暂存区**，显示“X 前”存入时间。
- 暂存区条目操作：
  | 操作 | 图标 | 说明 |
  |---|---|---|
  | **找回** | 点标题 | 重新打开该 tab，并**移出暂存区** |
  | **升级为收藏** | ⭐ | 从临时暂存提升为永久收藏（移入收藏夹） |
  | **彻底删除** | 🗑 | 从暂存区移除，不打开 |
- 数据存 `~/.tabboard_parked.json`，跨重启保留。

### 8. ⭐ 收藏夹（永久留存）

“长期参考、随时要找到”的文档用这个（如每周例会纪要、support doc）。是一个**可视化的智能收藏库**，替代传统文件夹式收藏夹。

- **进入**：点顶部「📚 收藏夹 (N)」按钮，切到收藏夹视图；按钮变「← 返回实时 tab」。
- **组织**：沿用同一套自动分类卡片（1:1 / 会议 / 教育 / PRD…），**可搜索**，一屏看全。无需建文件夹、无需打标签。
- **存**：在实时 tab 点 ⭐（不关闭）；或从暂存区点 ⭐ 升级。
- 收藏夹条目操作：
  | 操作 | 图标 | 说明 |
  |---|---|---|
  | **打开** | 点标题 | 在 Chrome 新开该链接，**保留在收藏夹**（不消失），并刷新“最后打开时间” |
  | **移除** | 🗑 | 从收藏夹删除 |
- 数据存 `~/.tabboard_library.json`，每项记录 `收藏时间` 与 `最后打开时间`。

### 9. 🧹 久未访问标记 & 周清理提醒

帮你定期清理收藏夹里长期没碰的，但**绝不自动删**——只挑出来让你决定。

- **久未访问标记**：超过 **30 天**没从收藏夹打开过的项，时间徽标变成橙色 **💤 N 天没点**。
- **周清理提醒**：进入收藏夹视图时，若**距上次清理 ≥ 7 天**且存在久未访问项，顶部弹出提醒条：
  > 🧹 收藏夹该清理了：**N** 项超过 30 天没点　[只看这些]　[知道了]
  - **只看这些** → 筛选成只显示久未访问项（顶部出现「← 显示全部收藏」切回），逐个查看、不要的点 🗑。
  - **知道了** → 关闭提醒，记下时间，7 天内不再弹。
- “上次清理时间”存 `localStorage`（key `tb_lastreview`）。

### 10. 自动备份 & 恢复

防止任何意外覆盖 / 误删 / 损坏。

- **每次写入**任一数据文件**之前**，自动把旧版本快照到 `~/.tabboard_backups/`，文件名带时间戳。
- 每个文件**保留最近 20 份**快照，超出的最老快照自动删除（文件很小，几十份仅几十 KB）。
- 覆盖收藏夹、暂存区、归类覆盖三类数据都会触发备份。

**从备份恢复**（万一需要）：
```bash
ls -a ~/.tabboard_backups/                                    # 查看所有快照（带前导点，用 -a）
cp ~/.tabboard_backups/.tabboard_library.json.<时间戳>.bak \
   ~/.tabboard_library.json                                   # 选一个时间点覆盖回去
# 然后重启 tab_board
```

## 数据存储与隐私

所有数据都是**纯本地、明文 JSON 文件**，在你的用户主目录 `~` 下，**不上传任何服务器、不进云端**：

| 文件 | 内容 |
|---|---|
| `~/.tabboard_library.json` | 收藏夹 `{url: {title, 收藏时间, 最后打开时间}}` |
| `~/.tabboard_parked.json` | 暂存区 `{url: {title, 暂存时间}}` |
| `~/.tabboard_overrides.json` | 手动归类覆盖 `{url: 分类key}` |
| `~/.tabboard_backups/` | 上述文件的带时间戳快照（保留最近 20 份/文件） |

浏览器端的**查看偏好**存在 `localStorage`（不离开浏览器）：

| key | 内容 |
|---|---|
| `tb_collapsed` | 哪些分组被折叠 |
| `tb_order` | 分组的自定义顺序 |
| `tb_lastreview` | 上次清理收藏夹的时间 |

> 设计原则：**数据**（收藏/暂存/归类，跨重启跨设备通用）放服务端文件；**查看偏好**（折叠/顺序/提醒时间，只属于这台机器这个浏览器）放 localStorage。

## 自定义分类规则

编辑 `tab_board.py` 顶部：

- **`CATS`**：分类的 key、emoji、名称、顺序（顺序即默认显示顺序）。
- **`RULES`**：`(分类key, [关键词...])`，顺序即优先级，命中 `标题+URL`（小写）任一关键词即归类。想新增一类，在 `CATS` 加一行、在 `RULES` 加对应关键词即可。
- **`HOST_RULES`**：按域名兜底归类。

改完重启（或 editable 安装下直接生效）。

## 已知限制

- **仅 macOS + Chrome**：依赖 AppleScript 控制本机 Chrome。
- **不做物理重排真实 tab 栏**：Chrome 的 AppleScript 移动 tab 会把页面重置为空白（丢登录态/滚动），有损。所以“聚拢同类”由看板的分组视图承担，不去动真实 tab 栏的顺序。
- **数据只在本机**：本地文件 + 本地备份，没有云同步（这也是隐私安全的代价）。
- **收藏的“最后打开时间”只统计从看板打开**：你直接在 Chrome 里访问该页面不会刷新它（影响久未访问判定，属可接受近似）。

## 常见问题

**Q：刷新后一片空白？**
顶部状态栏现在会直接显示 `⚠️ + 具体错误`（不再吞成空白）。最常见原因是有**多个 tab_board 进程残留**导致浏览器连到旧进程——用 `pkill -f tab_board.py` 全部杀掉，再启动一个即可。

**Q：怎么完全重新启动？**
```bash
pkill -f tab_board.py; sleep 1; python3 ~/Desktop/tabboard/tab_board.py
```

**Q：会不会偷偷打开/关闭我的 tab？**
所有开关 / 跳转 / 收藏 / 暂存动作都只在你点击时发生。后台只做只读的“读取 tab 列表”。

**Q：端口能改吗？**
改 `tab_board.py` 顶部 `PORT = 8765`。
